## Comment

La stack technique utilisée pour ce blog est relativement simple. A l'heure à laquelle j'écris cette article, voici les outils qui le font tourner:
- [NextJS v13](https://beta.nextjs.org)
- React Server Components
- [TailwindCSS](https://tailwindcss.com)
- [ContentLayer](https://contentlayer.dev)

**NextJS** et les **Server Components** vont très bien ensemble pour avoir un site dynamique tout en réduisant la quantité de JavaScript qui est envoyé au client. Pour le faire courte, les composents sont rendus côté serveur, et le retour est envoyé sous forme de HTML au client _(la vérité est un peu plus complèxe, mais je vous laisse chercher la réponse par vous-même)_

**TailwindCSS** n'a plus vraiment besoin d'être présenté, c'est un outils qui permet, plutôt que faire des aller-retours entre votre `JSX` et votre `CSS`, d'utiliser un système de classes utilitaires directement sur votre élément HTML. Par exemple:
```html
<!-- Avec Tailwind -->
<div class="px-4" />


<!-- En CSS pure -->
<div style="padding-left: 1rem; padding-right: 1rem;" />
```
_Il va de soit que dans le vrai monde, il faut créer une classe pour cette div, et non pas utiliser le inline-style_

Et enfin, **ContentLayer**, qui permet la gestion du contenu Markdown (ou MDX) sous stéroides!
Ce _SDK_ valide et transform le contenu en un JSON _type-safe_ et permet l'importation directement dans notre application sous la forme d'un object JS.
